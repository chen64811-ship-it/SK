## 一、为什么是这三个

从 Hippocratic AI 的底层结构出发——

```
高频关系动作 → AI自动触达 → 判断用户当前状态 → 推进下一步流程 → 遇到风险转人工 → 沉淀数据 → 优化下一轮触达
```

在医美/保险/康复/教培四个候选赛道中，筛选出三个最能回答关键问题的产品：

| 产品 | 回答的关键问题 |
|------|-------------|
| **Hippocratic AI**（原版·医疗） | 这个模式的原始定义者是怎么设计状态机、风险升级、定价模型的？ |
| **暖哇 阿拉莫斯**（中国·保险） | 跨行业迁移是否成立？免费部署背后的商业逻辑是什么？ |
| **岚时 AMAA**（中国·医美） | 在中国做垂直高频触达，SaaS+AI模式能否跑通？纯AI Worker有机会吗？ |

三个拆完，团队应该能判断：在中国做"高频关系运营AI员工"，应该选什么行业、用什么产品形态、怎么定价、怎么跟存量SaaS竞争。

## 二、Hippocratic AI（原版·医疗）

### 2.1 基本信息

| 维度 | 内容 |
|------|------|
| 产品名 | Hippocratic AI |
| 官网 | https://hippocraticai.com |
| 一句话 | 医疗机构的AI患者沟通员工——不当医生、不做诊断，只做预约提醒/出院随访/用药提醒/慢病跟进/患者教育 |
| 融资 | $141M Series B（a16z + General Catalyst 联合领投），累计融资约 $200M+ |
| 估值 | 未公开，推测 $500M-$1B |
| 客户 | 已有医院客户部署（具体名单未公开） |
| 证据等级 | **A**（多源：VC + 客户 + 媒体） |

### 2.2 它到底卖什么

**表面卖的是"AI医疗"——实际上卖的是三样东西的组合：**

| 卖点 | 说明 | 客户为什么买单 |
|------|------|-------------|
| **更便宜的劳动力** | 一个护士打随访电话的成本 vs AI打电话的成本 | 省钱（直接ROI） |
| **更稳定的质量** | 人类护士会忘、会烦、会漏，AI不会 | 少漏客户（隐性ROI） |
| **24小时在线** | 患者深夜问问题也有人回，术后焦虑随时响应 | 患者满意度（间接ROI） |

核心洞察：**Hippocratic AI 的客户买的不是"AI"，买的是"更便宜、更稳定、24小时在线的关系运营劳动力"。**

### 2.4 关键设计决策（最值得学的地方）

**决策1：只做"非诊断"**

- 它永远不说"你的疼痛可能是感染，需要吃抗生素"
- 它只会说"你的疼痛评分比昨天高了2分，我把这个情况转给王医生，她10分钟内联系你"
- **为什么聪明**：规避了医疗AI最大的监管和责任红线。在中国，这条线同样是医疗AI的死线。

**决策2：风险分层 = 核心护城河**

- 不是所有患者都一样重要。术后D+3疼痛评分6分的患者、伤口照片有红斑的患者，应该比"一切正常"的患者更早升级到人工。
- Hippocratic AI 的价值不在"能打电话"，在"知道什么时候不该AI打电话"。

**决策3：卖结果不卖AI**

- 提供给医院的数据是："本月我们自动完成了1247次随访，发现了83个需要医生介入的异常案例，推动了61个复诊预约"
- 不是"我们的模型在MMLU上达到XX分"。

**决策4：电话 > 文本**

- 美国医疗场景中患者更习惯电话（尤其是老年人），电话的互动率和信息收集质量远高于短信。
- 在中国，这个判断需要重新验证——可能是微信＞电话。

### 2.5 迁移到中国的关键问题

1. **交互方式**：美国用电话，中国可能用微信/企业微信/小程序。电话在中国医疗场景的接通率和信任度需要验证。
2. **数据接入**：美国医院有标准化的HIS/EMR接口，中国医院的系统碎片化严重得多（HIS、LIS、PACS各家用各家的）。
3. **付费方**：美国是医院直接付费，中国公立医院付费动力弱（DRG/DIP后可能有变化），私立医美/康复机构付费意愿更强。
4. **合规**：中国没有HIPAA但有《个人信息保护法》+《健康医疗大数据管理办法》，医疗数据的存储和处理有额外要求。

## 三、暖哇科技 阿拉莫斯（中国·保险）

### 3.1 基本信息

| 维度 | 内容 |
|------|------|
| 产品名 | 阿拉莫斯（Alamos）· 保险全流程AI用户经营产品 |
| 所属公司 | 暖哇科技（Nuanwa Technology） |
| 官网 | https://www.nuanwa.net |
| 子产品 | 语音智能体"天机" |
| 一句话 | 保险公司的AI续保员工——独立承担答疑、预报价、续保跟进，释放人工聚焦高价值客户 |
| 融资 | 连续六年入选中国保险科技100强（具体融资未公开） |
| 证据等级 | **A**（多源：客户落地 + 行业认证） |

### 3.2 它跟 Hippocratic AI 的结构对比

| 结构要素 | Hippocratic AI | 暖哇 阿拉莫斯 | 是否一致 |
|------|------|------|:--:|
| 高频触达 | 术后D+1/D+3/D+7随访 | 续保到期前30天/15天/7天/当天提醒 | ✅ |
| 状态判断 | 疼痛评分、伤口照片、用药依从 | 客户价值分层、沉默风险、续保意愿 | ✅ |
| 推进流程 | 康复指导、预约复查 | 预报价、异议处理、签约引导 | ✅ |
| 风险升级 | 异常症状→转医生 | 高价值/高流失风险→转人工坐席 | ✅ |
| 数据回流 | 患者依从性、复诊率 | 续保转化率、客户满意度、话术优化 | ✅ |
| 交互方式 | 电话为主 | 电话为主 | ✅ |

**结论：结构完全一致。Hippocratic AI 的底层模型可以跨行业成立。**

### 3.3 最值得深挖的设计决策

**决策1：语音智能体"天机"免费部署**

这是暖哇最反直觉的动作。一个AI语音产品，免费给保险公司用。

可能的商业逻辑（需要验证）：
- **数据飞轮**：免费部署→更多保险公司使用→更多真实对话数据→模型更准→产品更强→更多人用
- **保险佣金分润**：不是卖SaaS，而是参与续保佣金分成。如果AI能独立完成续保，暖哇从每一单续保中拿1-3%的分润。
- **增值服务收费**：免费的是基础版，高级版（定制话术、专属模型、数据分析报告）收费。
- **锁定效应**：免费部署后保险公司的人力、流程都在暖哇系统上，切换成本极高。

**决策2：双轨记忆体系**

- 短时记忆：当前对话的上下文（这个客户刚才说了什么）
- 长时记忆：客户的历史交互记录（这个客户上次续保时在纠结什么、他的家庭情况、他之前投诉过什么）
- **迁移价值**：这个设计直接适用于医美术后的"高频关系运营"——AI记得患者上次术后D+3说了什么，D+7的电话就不会像第一次打来一样。

**决策3：1.7秒超低延迟**

保险续保电话的场景中，如果AI停顿超过2秒，客户会觉得"对面是机器人"然后挂断。1.7秒是"人类正常思考时间"的上限。

**迁移价值**：做任何AI外呼/语音产品，延迟是第一体验指标，不是模型能力。

### 3.4 迁移到医美/康复的关键问题

1. **对话复杂度**：保险续保的对话相对标准化（确认信息→报价→答疑→成交），医美术后随访的对话更开放（"伤口有点红正常吗"、"我吃了海鲜有关系吗"），对模型的医疗知识要求更高。
2. **合规风险**：保险的错误（报错价）是经济损失，医疗的错误（没识别出感染信号）是人身伤害。这要求医美场景的风险升级机制要比保险更保守、更敏感。
3. **付费意愿**：保险公司的IT预算远高于医美机构。卖给保险公司可以免费部署+佣金分成，卖给医美机构可能需要更简单的定价（按患者数/按通话数）。

## 四、岚时科技 AMAA（中国·医美）

### 4.1 基本信息

| 维度 | 内容 |
|------|------|
| 产品名 | AMAA 医美业务智能体 |
| 所属公司 | 岚时科技（成立于2017年） |
| 一句话 | 医美机构的AI客户管理员工——自动分群、术后回访、营销任务派发、话术监督 |
| 客户规模 | 5000+ 医美/消费医疗机构 |
| 证据等级 | **B**（客户验证，无公开融资/收入数据） |

### 4.2 产品结构中值得深挖的点

**特征1：它是"SaaS + AI模块"，不是"原生AI员工"**

岚时起家是医美SaaS（排班、库存、会员管理），AI是后来加上的。这意味着：
- **优势**：已有5000+存量客户，AI模块可以直接卖给现有客户，获客成本为零。
- **劣势**：AI模块需要迁就原有SaaS架构，可能不是最优的AI体验。比如术后回访的触发规则可能写死在SaaS里，而不是AI自主判断。

**核心问题**：如果做一个"只做术后随访"的纯AI Worker，能不能跟岚时竞争？

**推演**：
- 如果机构已经在用岚时的SaaS，切换成本高，纯AI Worker很难打破
- 但如果机构只想要"术后随访AI"，不想换整个SaaS，纯AI Worker就有机会
- 关键在于：术后随访这个动作，是否需要跟SaaS的其他模块（预约、库存、会员）深度耦合？

**特征2：AI智能工牌 V2.0 的语音语义分析**

这是岚时又一个"不是AI员工，但是AI增强了员工"的产品——它录下咨询师跟客户的对话，用AI分析话术质量，自动给出改进建议。

**值得学习**：它暗示了一个机会——在医美场景中，AI不一定替代人，可以先"增强人"（监督咨询师话术→发现问题→AI自动补位）。

**特征3：客户管理的状态机应该是隐式的**

岚时的产品描述里没有明确的状态机设计（"术后D+几做什么"），而是"自动标签分群→回访机制→售后任务派发"。这可能是它的弱点——**状态机设计不够显性，导致触达节奏不精准**。

**如果是纯AI Worker，状态机应该是这样**：
```
术后D+1: AI电话确认客户已到家，提醒冰敷，询问疼痛评分
  ├─ 疼痛≤3 → 正常，D+3再联系
  └─ 疼痛≥7 → 转人工护士，标记"高危"
术后D+3: AI微信发伤口护理视频，询问是否按时用药
  ├─ 已用药 → 点赞，D+7再联系
  └─ 未用药 → 追问原因，记录不依从风险
术后D+7: AI电话提醒拆线/复查，推送预约链接
  ├─ 已预约 → 完成本轮随访
  └─ 未预约 → D+1再次提醒，D+3转人工
```

### 4.3 跟岚时竞争的可能路径

| 路径 | 做法 | 可行性 |
|------|------|:--:|
| **API插件模式** | 不做SaaS，只做AI随访Worker，通过API对接各种医美SaaS的预约/会员数据 | 中 |
| **微信小程序** | 在微信生态做轻量AI随访，机构用小程序配置随访计划，患者在小程序/微信接收 | 高 |
| **白标给SaaS厂商** | 把AI随访能力打包成白标方案，卖给岚时这样的SaaS厂商 | 高 |
| **自建SaaS** | 做完整的医美SaaS+AI | 低（岚时已有5000+客户，新SaaS获客极难） |

## 五、三产品横向对比

### 5.1 核心维度对比

| 维度 | Hippocratic AI | 暖哇 阿拉莫斯 | 岚时 AMAA |
|------|:--:|:--:|:--:|
| **行业** | 医疗（医院） | 保险 | 医美 |
| **产品形态** | 原生AI员工 | 原生AI员工 | SaaS + AI模块 |
| **交互方式** | 电话 | 电话 | 电话+微信+SaaS |
| **状态机显性度** | 高（明确的术后天数节奏） | 高（明确的续保到期节奏） | 中（标签+任务，非时间轴驱动） |
| **风险升级** | 明确（异常症状→医生） | 明确（高价值/高风险→人工坐席） | 模糊（任务派发，非状态驱动升级） |
| **定价模型** | 推测按患者数/通话数 | 免费部署+可能的佣金分成 | SaaS年费+AI模块加价 |
| **中国落地** | 无 | ✅ 已落地 | ✅ 已落地 |
| **小团队可验证** | ❌（$200M+融资） | ❌（保险门槛高） | ⚠️（5000+客户护城河） |
| **对中国迁移的价值** | 学结构设计 | 学商业模式 | 学落地路径 |

### 5.2 同一个"高频关系运营AI员工"在不同行业的表现

| 结构要素 | 医疗（Hippocratic） | 保险（暖哇） | 医美（岚时/机会） |
|------|------|------|------|
| **触发时机** | 手术日期 | 保单到期日 | 治疗日期 |
| **触达频率** | D+1/3/7/14/30 | 到期前30/15/7/1天 | D+1/3/7/14/30/90 |
| **核心指标** | 并发症检出率、复诊预约率 | 续保转化率 | 复购率、异常检出率 |
| **风险信号** | 疼痛↑、红肿、不依从 | 沉默、历史投诉、比价 | 术后异常、沉默、竞品接触 |
| **升级对象** | 主治医生 | 人工坐席/保险顾问 | 咨询师/护士 |
| **数据资产** | 患者康复曲线 | 续保预测模型 | 客户生命周期价值 |

## 六、团队迁移决策框架

|------|:--:|:--:|:--:|:--:|

| 监管风险 | ⭐⭐⭐⭐（轻） | ⭐⭐⭐（中） | ⭐⭐（重） | ⭐⭐⭐⭐（轻） |

### 6.2 做什么产品形态？

| 形态 | 适合场景 | 风险 |
|------|------|------|
| **原生AI Worker**（Hippocratic/暖哇模式） | 行业有明确的"高频触达时间轴"、触达动作相对标准化 | 获客难（需要教育市场） |
| **SaaS + AI模块**（岚时模式） | 行业已有成熟SaaS、AI作为增值功能 | AI体验被SaaS架构限制 |
| **白标/API插件** | 行业有多个SaaS平台、机构不愿换系统 | 依赖合作伙伴的分发能力 |

### 6.3 第一版做什么具体动作？

建议遵守文件里的原则：**第一版不做平台，只做一个具体动作。**

| 候选动作 | 为什么选 |
|------|------|
| 医美术后7天随访 | 频率最高、最标准化、ROI最直接（一次手术=至少7次触达机会） |
| 医美复购召回（玻尿酸/肉毒素补打提醒） | 时间节点明确（玻尿酸6-12个月代谢）、客单价高、客户不反感 |
| 保险续保提醒+预报价 | 流程标准化、有明确到期日、暖哇已验证需求 |

### 6.4 怎么定价？

三个参考模型：

| 模型 | 来源 | 做法 | 适合谁 |
|------|------|------|------|
| 按结果分成 | 暖哇推测模型 | 免费部署，从续保/复购佣金中分成 | 有明确交易闭环的行业 |
| 按触达量 | Hippocratic推测模型 | 按患者数/通话数次收费 | B端客户，触达量大 |
| 按SaaS年费 | 岚时模型 | 基础SaaS年费+AI模块加价 | 有存量SaaS客户基础 |

### 6.5 怎么跟存量SaaS竞争？

| 存量SaaS的弱点 | 纯AI Worker的机会 |
|------|------|
| AI是"附加功能"，体验不是最优 | 原生AI体验，状态机精准驱动 |
| 数据锁在系统里，迁移成本高 | 做轻量化（微信小程序/API），不要求替换SaaS |
| 功能大而全，术后随访只是一个小模块 | 只做术后随访这一件事，做到极致 |
| 按年收费，机构感知不到直接ROI | 按结果收费，机构只为"多来的复购"付费 |

### 6.6 最可能失败的地方

Hippocratic AI 分析文件中已经列出了四条死法，翻译成中国语境：

| 死法 | 中国特别版 |
|------|------|
| "以为机构都需要AI" | **中国特别版**：医美机构老板口头说"我需要AI"，但只为"这个月多来了几个老客"付费。ROI必须量化到具体金额。 |
| "碰诊断、用药、医疗建议" | **中国特别版**：在中国做医疗AI，没有"非诊断"这个安全区概念。一旦AI说了"建议你涂XX药膏"，就是非法行医。约束必须硬编码。 |
| "每个客户都定制" | **中国特别版**：每家医美机构的术后护理方案不一样，老板会说"我们和别家不一样"。如果每单都定制，三个月就会变成AI外包公司。必须标准化80%。 |
| "LLM+话术，没有流程" | **中国特别版**：只接入DeepSeek API + 一套随访话术模板，没有状态机、没有跟HIS/CRM的数据对接、没有风险升级机制。这种产品会被任何有渠道的CRM公司三个月内复制。 |

### 7.2 拆解后应产出的判断 ✅

> 以下5个判断基于三个10维拆解的交叉验证。

**判断1：最小可行状态机**

```
触发层: 治疗日期/手术日期/保单到期日（从SaaS/系统API读取）
   ↓
节奏层: D+1/D+3/D+7/D+14/D+30/D+90（根据行业和项目类型配置）
   ↓
触达层: AI微信/电话触达（优先微信，电话做降级）
   ↓
对话层: 按协议问标准化问题 + 双轨记忆调取历史上下文
   ↓
判断层: 正常→推进下一节点 / 异常→转人工 / 沉默→加强触达
   ↓
回流层: 每次触达结果回写→更新客户状态→优化下一轮
```

**判断2：第一版选医美**

| 维度 | 医美 | 教培 |
|------|:--:|:--:|
| ROI清晰度 | ⭐⭐⭐⭐⭐（复购客单价5000-50000元） | ⭐⭐⭐（续费客单价5000-20000元） |
| 标准化程度 | ⭐⭐⭐⭐（术后随访有明确临床路径） | ⭐⭐⭐（课程差异大、进度不一） |
| 监管风险 | ⭐⭐⭐⭐（非诊断，不碰医疗建议） | ⭐⭐⭐⭐⭐（几乎无监管） |
| 私域成熟度 | ⭐⭐⭐⭐⭐（咨询师微信+客户群成熟） | ⭐⭐⭐（依赖班主任个人微信） |
| 付费用力 | ⭐⭐⭐⭐⭐（老板能直接算清"多一个复购=多5000"） | ⭐⭐⭐（续费是长期效果，难量化到单次触达） |

**判断3：第一版做微信小程序 + 微信触达，不做电话**

原因：
- 中国医美消费者以25-40岁女性为主，微信使用率几乎100%，电话接通率低（陌生号码不接）
- 微信消息可附带图片/视频/预约链接（电话只能语音）
- 微信对话记录天然形成"双轨记忆"的数据源
- 电话外呼需要额外的基础设施投入（通信资质+语音引擎），微信触达零成本启动
- 但需要保留"电话降级"能力——如果客户D+3未回微信，AI自动拨电话

**判断4：按结果分成，不按SaaS年费**

| 定价模型 | 为什么选/不选 |
|---------|-------------|
| SaaS年费（如岚时1800元/年） | ❌ 天花板太低，无法体现结果价值 |
| 按触达量（如每条随访0.5元） | ⚠️ 简单但不性感——客户会算"我雇人打电话也差不多" |
| **按结果分成（如复购金额的1-3%）** | ✅ 客户只为"多来的复购"付费，ROI不需要解释。一个术后随访带来的复购客单价5000-50000元，1-3%=50-1500元/单。一个年营收500万的机构，AI带来的复购增量约75-150万（15-30%提升），服务费约1.5-4.5万/年——是岚时SaaS年费的10-25倍 |

**判断5：跟岚时/领健竞争的正确姿势**

不做SaaS，做"通过API对接SaaS的纯AI Worker"：
- 机构不需要换SaaS（岚时/领健/有赞的客户都能用）
- AI Worker从SaaS读预约和治疗数据，自动执行术后随访
- 随访结果回写SaaS + 推送给咨询师的微信
- 咨询师不用登录新系统——她在微信里就能看到"AI标记了3个异常客户"

## 八、最终产出：三个可执行交付物

### 交付物①：底层结构模板——"高频关系运营AI员工"六步结构

> 来源：Hippocratic AI的状态机设计 + 暖哇的双轨记忆 + 岚时的"不做SaaS做Worker"策略

```
┌─────────────────────────────────────────────────────┐
│           高频关系运营AI员工 · 底层结构模板             │
├─────────────────────────────────────────────────────┤
│                                                       │
│  Step 1 触发 ── 从行业数据源读取触发时机               │
│  ├─ 医疗: 手术日期 (HIS/EMR)                          │
│  ├─ 保险: 保单到期日 (核心系统)                        │
│  ├─ 医美: 治疗日期 (SaaS API)                         │
│  └─ 教培: 课程周期 (教务系统)                          │
│                                                       │
│  Step 2 触达 ── AI自动发起触达（优先微信/降级电话）     │
│  ├─ 微信/企微消息（文字+图片+视频+链接）               │
│  ├─ 电话外呼（语音对话+自然插话+<1.7秒延迟）           │
│  └─ 短信/App推送（辅助提醒）                           │
│                                                       │
│  Step 3 判断 ── 基于标准协议判断用户当前状态            │
│  ├─ 正常: 按节奏推进下一步                             │
│  ├─ 异常: 触发升级标记                                 │
│  └─ 沉默: 加强触达频率或切换触达渠道                   │
│                                                       │
│  Step 4 推进 ── 根据状态驱动下一步流程                  │
│  ├─ 正常→发送下一节点的标准内容                        │
│  ├─ 异常→打包上下文+异常标记→推送给人工                │
│  └─ 沉默→D+1自动重新触达                               │
│                                                       │
│  Step 5 升级 ── 遇到风险/异常转人工                     │
│  ├─ 明确的风险阈值（疼痛>6/伤口异常描述/投诉倾向）      │
│  ├─ 自动打包上下文（历史对话+本次异常+建议动作）        │
│  └─ 推送至人工（咨询师微信/坐席系统）                   │
│                                                       │
│  Step 6 回流 ── 沉淀数据→优化下一轮                     │
│  ├─ 双轨记忆更新（事实记忆+状态记忆）                   │
│  ├─ 状态机节奏优化（哪些节点触达效果最好）              │
│  └─ ROI数据看板（本月AI独立完成XX次触达/发现XX异常/推动XX复购）│
│                                                       │
└─────────────────────────────────────────────────────┘
```

**这个模板的三个不可省略的组件**（从三个产品交叉验证得出）：

| 组件 | 来源 | 为什么不可省略 |
|------|------|-------------|
| **双轨记忆**（事实+状态） | 暖哇阿拉莫斯 | 没有它，AI每次都是"第一次认识你"——高频触达的前提是"记得" |
| **风险升级机制** | Hippocratic AI | 没有它，AI会漏掉真正需要人介入的客户——这是"非诊断"定位的核心保障 |
| **按结果定价** | 岚时的反面+暖哇的正面 | 卖SaaS赚不到钱（岚时1800元/年），卖结果可以（复购分成1-3%=1-5万/年） |

### 交付物②：行业选择判断——为什么第一版是医美

三个产品拆完后，最核心的判断是：**第一版选医美，不选保险。**

不是因为保险不好（暖哇证明了保险场景成立），而是因为：

| 关键约束 | 保险赛道 | 医美赛道 |
|---------|:--:|:--:|
| 小团队可进入 | ❌（保险公司的采购周期6-12个月，需要BD团队） | ✅（单体诊所老板一个人就能决策，当天试用当天上线） |
| 数据获取难度 | ❌（保险公司的核心系统不对外开放） | ✅（通过API对接现有SaaS即可获取预约和治疗数据） |
| 竞争烈度 | ❌（暖哇已IPO+头部险企自研） | ⚠️（岚时/领健在做SaaS，但没人在做"纯AI执行"） |
| 付费意愿 | ✅（保险公司的IT预算远高于医美机构） | ✅（机构老板能直接算清"一个复购=多少钱"，ROI不需要解释） |
| 合规风险 | 中（保险销售合规，但不涉及人身安全） | 低-中（只要守住"非诊断"边界） |

**判断：医美是小团队能在6个月之内跑通第一个付费闭环的赛道。保险需要更大的团队、更多的资金、更长的销售周期。**

### 交付物③：第一版MVP定义——"医美术后随访AI Worker"

**MVP一句话**：
一个微信小程序，医美机构扫码绑定后，AI自动从现有SaaS读取治疗记录，按术后D+1/3/7/14/30标准节奏自动给客户发微信做术后随访，异常客户实时推给咨询师。

**MVP的具体边界**：

| 维度 | 做 | 不做 |
|------|----|------|
| **项目覆盖** | 水光针、玻尿酸、热玛吉（3个最常见轻医美项目） | 整形手术（对话复杂度更高，先不做） |
| **触达方式** | 微信模板消息+微信客服消息 | 电话外呼（第二版再加） |
| **数据对接** | 手动上传Excel + 1个SaaS API对接（优先岚时/领健） | 全量API对接（先验证需求再工程化） |
| **状态机** | 术后D+1/3/7/14标准节奏，3个项目的随访协议模板 | 全项目覆盖+自定义节奏 |
| **记忆系统** | 基于微信聊天记录的单客户上下文记忆 | 双轨记忆的完整实现（第二版） |
| **风险升级** | 关键词触发升级（"红肿""化脓""特别疼""发热"） | AI语音情绪识别（第二版） |
| **定价** | 前50个患者/月免费，超额按复购金额1-3%收费 | SaaS年费模式 |
| **目标客户** | 单体诊所/医生创业店（没有专职咨询师的） | 大型连锁（先单点跑通再扩展） |

**MVP的验证指标**（3个月内）：
1. 10家付费机构
2. 术后随访覆盖率 > 95%（对比人工的60-80%）
3. 异常检出率 > 80%（人工能发现的，AI不遗漏）
4. 复购率提升 > 10个百分点（有AI随访 vs 没有AI随访）
5. 月收入 > 5万（证明付费模型成立）
