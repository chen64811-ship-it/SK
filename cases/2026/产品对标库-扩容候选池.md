## 一、为什么新增这个文件

原 `产品对标库-38个AI产品复制价值排名.md` 已降级为档案，不再适合作为动态新增产品池。

本文件用于承接新信号：

信号源 → 扩容候选池 → 轻量初筛 → A/B/C/D 分级 → 是否进入正式产品对标库 → 是否进入轻量初拆 / 标准 10 维深拆

原则：

- 不恢复旧产品雷达周报机制；
- 不把所有发现都塞进正式对标库；
- 不做收藏夹；
- 只记录可能影响 SK 判断力和内容生产的候选产品；
- 每新增 3 个成功/机会候选，至少同步补 1 个失败候选到 `失败案例候选池.md`。

## 八、轻量初筛五问

每个候选产品最多 5 分。

| 问题 | 1 分标准 |
|---|---|
| 1. 是否有明确用户痛点？ | 能说清"那个卡住的人" |
| 2. 是否有可核验证据？ | 官网外至少 1 个外部来源，或真实用户反馈 |
| 3. 是否有中国迁移空间？ | 中国有相似痛点，且现有替代路径不充分 |
| 4. 是否适合小团队先验证？ | 2–5 人能做 MVP、内容验证或访谈验证 |
| 5. 是否有反直觉机制？ | 不只是"AI 加某行业"，而是改变流程、成本、信任、速度或分发 |

分级：

| 分数 | 处理 |
|---:|---|
| 5 | 进入正式产品对标库候选 |
| 4 | 进入轻量初拆 |
| 3 | 观察 |
| 1–2 | 仅留线索 |
| 0 | 删除或归档 |

## 九、进入正式产品对标库的升级规则

候选产品进入正式对标库前，必须满足：

1. 至少 2 个独立信号源；
2. 至少 1 个可核验外部 URL；
3. 能写出"那个卡住的人"；
4. 能初判 A/B/C/D；
5. 能说明中国机会不是纯想象；
6. 不与已拆产品重复，或能说明新增差异。

## 十、放弃 / 归档规则

以下情况直接放弃或仅留观察：

1. 只有官网宣传，无外部证据；
2. 只是大模型套壳，没有工作流变化；
3. 中国已有强替代且无差异化；
4. 小团队无法验证；
5. 监管、牌照、供应链门槛过高；
6. 只有融资热度，没有用户痛点；
7. 无法说清"谁卡住了"。
